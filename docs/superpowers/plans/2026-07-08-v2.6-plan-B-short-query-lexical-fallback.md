# trelix v2.6.0 — Plan B: Short-Query Lexical Fallback

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the verified product gap where all embedding models collapse to near-zero nDCG@10 (0.000–0.015) on short developer keyword queries (≤5 tokens), by routing such queries to BM25-first retrieval before embedding.

**Architecture:** Research finding (CoREB arXiv:2605.04615, confirmed 3-0): all evaluated models score near-zero on short keyword queries — the format most common in real developer search. The fix is a pre-routing layer in `AdaptiveRouter` that detects short queries (≤5 meaningful tokens after stop-word removal) and sets `bm25_tokens_only=True` on the SubQuery, which tells `_run_subquery_legs` to skip vector ANN and run BM25 + grep only. This is the "lexical fallback for short queries" pattern — pure lexical search outperforms embedding at this query length because embeddings compress short queries into semantically ambiguous vectors. The gate is opt-in via `TRELIX_RETRIEVAL_SHORT_QUERY_LEXICAL=true` (default `false`) so users can A/B test it and it won't break existing behavior.

**Tech Stack:** Python 3.11+, existing `bm25.py` (already has stop-word filtering), existing `AdaptiveRouter`, existing `RetrievalConfig`. No new dependencies.

```python
def count_meaningful_tokens(query: str) -> int:
    """
    Count non-stop-word tokens of length ≥ 3 in `query`.

    Uses the same tokenization as `_preprocess_query` — splits on
    word-character boundaries, drops stop words, drops tokens < 3 chars.
    This is the signal for short-query detection.
    """
    raw_tokens = re.findall(r"[a-zA-Z_][a-zA-Z0-9_]*", query)
    return sum(
        1 for t in raw_tokens
        if t.lower() not in _STOP_WORDS and len(t) >= 3
    )


def is_short_query(query: str, threshold: int = 5) -> bool:
    """
    Return True when `query` contains ≤ threshold meaningful tokens.

    Short queries (≤5 meaningful tokens after stop-word removal) perform
    near-zero with embedding models across all code retrieval benchmarks
    (CoREB arXiv:2605.04615). Routing them to BM25+grep lexical search
    significantly improves recall for developer keyword-style queries.

    Args:
        query:     Raw user query string
        threshold: Maximum meaningful-token count to classify as "short"
                   (default 5, configurable via TRELIX_RETRIEVAL_SHORT_QUERY_TOKENS)
    """
    return count_meaningful_tokens(query) <= threshold
```
