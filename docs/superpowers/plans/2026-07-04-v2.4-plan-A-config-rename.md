# v2.4 Plan A — flare_max_iterations → flare_max_retries (Backward-Compatible Rename)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename `flare_max_iterations` → `flare_max_retries` in `RetrievalConfig` so the semantics match the default (`default=1` means 1 retry, not 0 iterations), while accepting both `TRELIX_RETRIEVAL_FLARE_MAX_ITER` (old) and `TRELIX_RETRIEVAL_FLARE_MAX_RETRIES` (new) env vars with a `DeprecationWarning` for the old name.

**Architecture:** pydantic-settings v2 `AliasChoices` accepts multiple env var names for a single field; `validation_alias` covers input-only (no serialization side-effects). A `model_validator(mode='after')` checks `os.environ` for the old name and emits a `DeprecationWarning`. The field rename from `flare_max_iterations` to `flare_max_retries` is reflected in all callsites.

**Tech Stack:** pydantic>=2.7.0, pydantic-settings>=2.3.0 (both already in pyproject.toml).

Open `src/trelix/core/config.py`. Find the `flare_max_iterations` field (around line 429). Replace it with:

```python
    flare_max_retries: int = Field(
        default=1,
        ge=1,
        le=3,
        validation_alias=AliasChoices(
            "TRELIX_RETRIEVAL_FLARE_MAX_RETRIES",
            "TRELIX_RETRIEVAL_FLARE_MAX_ITER",  # legacy — deprecated in v2.4
        ),
    )
```

Also add the `model_validator` to `RetrievalConfig` (add after the field definitions, before any existing validators):

```python
    @model_validator(mode="after")
    def _warn_deprecated_flare_iter_env(self) -> "RetrievalConfig":
        import os
        import warnings

        if "TRELIX_RETRIEVAL_FLARE_MAX_ITER" in os.environ:
            warnings.warn(
                "TRELIX_RETRIEVAL_FLARE_MAX_ITER is deprecated as of trelix v2.4.0. "
                "Use TRELIX_RETRIEVAL_FLARE_MAX_RETRIES instead. "
                "The old name will be removed in v3.0.0.",
                DeprecationWarning,
                stacklevel=2,
            )
        return self
```
